Android的碎片化

​ 众所周知，Android的碎片化问题已经不是一天两天的事情了，系统的碎片化，屏幕的碎片化，导致Android的发展方向如何…. 聊跑题了，话说回来哈，其实最为严重的就是屏幕的碎片化，截止到2014年就已经有18796中屏幕尺寸了，这是要干嘛？不过碎片又能如何，有我适配大法，天下任我行，哈哈哈….

乱中找规律

​ 通过各大权威统计机构统计，市面上主要运行的手机机型的分辨率无非就六种（1280 720、1920 1080、854 480、800 480、 960 540、1184 720），所以我们在屏幕适配之前先不要慌，要先确认我们产品需要适配那种机型，然后在具体的机型操作，在这里我们需要注意三种机型的存在。

a. 1920 1080 目前最大分辨了的机型（5.0英寸），这将是手机分辨率的趋势，所以这种机型势必是要做适配的
b. 1280 720 这种机型是UI们最喜欢的机型，大部分UI会用这种分辨率来做基准进行切图和做效果图的
c. 480 320 这是Android的dp与px的基线分辨率，这个之后会有详解

适配原理

这个是一个基础，经常面试官会问到，实际开发中几乎不会用到，不过我们需要了解。

先上几个屏幕适配的概念：

1.屏幕像素密度：每英寸上的像素点数，单位dpi
2.屏幕分辨率：手机屏幕的像素点数，单位px
3.屏幕尺寸：手机对角线的大小，单位英寸
4.dp：android特有的 是一种密度计算，单位dip
5.sp：android用来规范字体大小的单位，可以在系统调整字体大小是发生变化

再说说dp与px的对应关系

1.首先我们需要算出像素密度（dpi）
​ dpi = 根号下 高的平方 + 宽的平方 / 屏幕尺寸
2.然后有一个Android规定的对应表，这里以宽为例

​ 320 240 120 1dp = 0.75px
​ 480 320 160 BaseLine 1dp = 1px
​ 800 480 240 1dp = 1.5px
​ 1280 720 320 1dp = 2px
​ 1920 * 1080 480 1dp = 3px

3.对应android工程下的dpi文件夹
​ ldpi 120
​ mdpi 160
​ hdpi 240
​ xhdpi 320
​ xxhdpi 480
​ xxxhdpi 640

解决方案

1.布局适配

​ 这种方式基本不会用到，除非做平板适配，应为这种需要些多套布局，太耗费开发成本。

2.权重适配

​ 在写布局时，能使用这种方式尽量就要使用这种方式，但是这种方式有一个局限，就是只能在线性布局中使用，简单说一下权重适配的原理：自身的大小 + 占剩余空间的比例

3.图片适配

​ 不同分辨率图片要放在对应大小的dpi文件夹下，否则因为android读取图片文件的机制导致，加载图片文件是的使用内存，会成倍数增长。纯色图片可以使用.9图的方式进行图片适配。

4.代码动态适配

​ 这种方式经常会在，需要代码中设置px的值时使用。简单原理就是，拿到屏幕的宽高，然后动态的改变控件的布局。

5.限定符适配

​ Android规定了land、port、small等一系列的限定符，这种限定符的作用就是在，android系统可以在初始化应用是，自动的去适配相应限定符文件下的资源文件。

6.dimens适配

​ 这个在下边着重进行讲解，这种方式是一定要用的方式。

Dimens适配

​ 看了好多博客，感觉hongyang博客写的不错可以借鉴http://blog.csdn.net/lmj623565791/article/details/45460089。

​ 大概的方式是这样的，写一个java类，里边写着换算逻辑，执行一下，生成一堆dimens文件，然后我们在写不布局文件时，不写具体的dp，而是根据UI的效果图基准分辨率找对应文件夹的下像素对应的dimens名。
