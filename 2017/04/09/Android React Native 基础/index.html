React Native

React Native 组件

在RN项目中，所有展示的界面都可以看成是一个组件（Component），根据功能和逻辑上复杂度的不同，不同大大小小的组件有不同的逻辑代码，不过它们都遵循同样的代码结构，都由一下几部分组成。

1.包和其他组件的引用部分
这一部分利用node.js的require机制来引入react-native或者其他node.js包，以及引入其他第三发api和组件等。

2.组件类的声明
这一部分用于构造组件的状态和具体的展示结构，其中render属性尤为重要，用于渲染整个组件的结构和布局。其中组件中重要的生命周期也发生在一部分。

3.组件样式的声明
这一部分通过StyleSheet.create来实例化样式对象，其中的内容为Json形式的React Native样式。样式规范来源于CSS3。

ReactNative组件生命周期

一个RN组件的生命周期分为实例化（mount）、存在期（update）和销毁期（unmount），其中最为常用的是实例化期，该时期包含组件的构建和展示。需要根据几个生命周期的方法控制好组件的展示和逻辑的处理。

1.实例化时期（5个阶段）
getDefualtProps()————————实例化时调用之后不会再调用，用来初始化固定值，如静态数据源
getInitialState()—————————初始化状态值，可用于改变组件状态
componentWillMount()—————–组件将被加载前调用
render()————————————-组件的渲染（尤为重要）
componentDidMount()——————组件已加载、渲染完成

注：render方法通过return来返回一个jsx，因为是通过DOM解析布局，所以只能返回一个顶级元素，componentDidMount方法即执行hook函数，一般用来网络数据的异步加载，开发时可以通过fetch API来进行网络请求。

2.存在期
一般用来处理用户交互，如：点击事件等。（并不是很常用）
componentWillReceiveProps()———-指父元素对组件的props或state进行了修改
shouldComponentUpdate()—————一般用于优化，可以返回false或true是否进行渲染
componentWillUpdate()——————-组件刷新前调用，类似于componentWillMount方法
componentDidUpdate()——————–更新后的hook

3.销毁期
用于清理一些无用的内容，如：点击事件的listener，只有一个方法：componentWillUnmount()
