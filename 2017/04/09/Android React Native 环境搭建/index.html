Android React Native 环境搭建

发表于 2017-04-09 | 分类于 React Native
火的不行不行的React，我是不是也应该蹭一把热度，不过话说回来，这种开发模式确实在开发周期上优势明显。

JDK SDK Node.js Git React Native Gradle Curl

话不多说直接开搞！come on ！

Windwos下环境搭建

1.安装JDK，我觉得这一步也不用多说了吧，注意x86的、版本1.7、配置环境变量
2.下载SDK，这一步我也不想多说啥了，注意配置环境变量（环境变量名称为ANDROID_HOME）
3.下载node.js,这里我还是不想多说，到官网下载4.1版本以上就行，然后安装
4.下载git，这个就是更easy了，好了，都下好安好了，开始搞起
5.下载facebook的react吧

方法一：直接使用npm方式，执行npm install -g react-native-cli

方法二：使用git方式
a. 在git里边： git clone https://github.com/facebook/react-native.git
b. 在cmd里边：cd react-native/react-native-cli
c .npm install -g

6.创建项目 react-native init 项目名称，在初始化之前最好使用国内的镜像路径，因为有很多包是国外的，在安装是会因为网络超时或者访问不到而挂掉，改变镜像路径有两种方式。

方法一：
在~\Roaming\npm\etc\npmrc中写入镜像
registry=https://registry.npm.taobao.org/
disturl=https://npm.taobao.org/dist

方法二：
执行下边的命令
npm config set registry https://registry.npm.taobao.org
npm config set disturl https://npm.taobao.org/dist

7.开启服务 react-native start
8.打开Android Studio，找到本地的RN工程中的android下边的build.gradle然后打开这个RN中产生的Android项目
9.开启也可以在开启模拟器的情况下，执行react-native run-android

RN调用Android原生模块方法

1.用AS打开RN中的Android项目
2.在Android这边创建一个继承PreactContextBaseJavaModule的自定义模块，这个类中存放RN需要调用的方法，其中getName()方法中需要设置一个这个模块对外部的名字，这个是用来在RN中进行调用的，然后我们这里创建的方法需要加上@ReactMethod注解，这个是为了将这个方法暴露给RN
3.在Android这边创建一个实现ReactPackage接口的包管理器，这类中需要实现接口的三个方法我们只需要在创建原生模块方法中创建一个模块列表，并将上一步创建的自定义模块添加到这个列表中就可以了，其他两个方法返回Collections.emptyList()就可以了
4.在Android这边的MainApplication的getPackages()方法中，将上一步创建的包管理器添加到asList列表中就可以了，这个是为了加载我们自定义的包管理器及其其中的自定义模块
5.在RN这边添加facebook提供的NativeModules到import中，然后使用onPress={this.点击事件名字.bind(this)}，最后在点击事件中添加NativeModules.自定义模块.暴露的方法()，这样就实现了RN调用Android原生的方法了。

3.上一步已将android基本配置完成，下面进行index.android.bundle的生成

a.下载curl.exe，这个软件可以在官网下载，下载Win64 x86_64 CAB版本的，然后配置环境变量

c.测试是否安装成功，cmd黑窗口执行curl –help

5.在AS工程的Proguard做一些防止混淆的操作，防止工程中的一些第三方包被混淆了，导致应用挂掉， 其中enableProguardInReleaseBuilds 换为true，这个换不换都行
6.打包生成apk，很抱歉这一步也不能像常规app一样，所以只能敲命令喽

a.下载或者你本地的as中有gradle，那就直接配置环境变量就行和第3步大同小异，不在重复，可以在黑窗口中执行gradle -v来确认是否安装成功
b.安装gradle之后，在项目工程/android/目录下执行gradle clean 清理项目中安装包，然后再执行gradle assembleRelease命令生成已签名的apk，这个apk会在android/app/bulid/outputs/apk/目录中。
