博客搭建

发表于 2017-03-09 | 分类于 Blogger
这是一款基于GitHub做服务器，搭载Hexo个人博客框架，采用自己域名的博客系统。

Git GitHub Node.js Hexo Next

话不多说直接开搞！come on ！

环境搭建

创建GitHub并在其中创建仓库

安装Node.js和安装Git，这里就话不多说。

为自己的GitHub配置SSH秘钥

a.进入到本机所在的.ssh文件夹下，大概位置“C:\用户\你的用户名\.ssh”
b.然后在这个文件夹下打卡Git黑窗口，执行$ ssh-keygen -t rsa -C "你的Github邮箱" 命令。
c.输入密码，这个过程是为了防止你即使丢了秘钥别人也不能直接修改你的GitHub仓库
d.秘钥生成成功之后，可以在本地的../.ssh目录下找到一个`id_rsa.pub`这个文件 ，然后打开复制全部代码。
e.接下来打开GitHub，选择`Settings——SSH and GPG keys——New SSH key`，将我们上一步复制的代码粘贴到key中，title可以自定义。
f.填写完成之后，选择`Add SSH key`按钮。

本地Hexo的创建

a.打开Git黑窗口安装hexo， $ npm install hexo -g
b.创建一个博客文件夹，目前这个文件要求是一个空文件夹
c.执行$ hexo init，初始化Hexo
d.执行$ hexo g，生成静态页
e.执行$ heso s，启动本地服务
到这一步，我们打开本地 http://localhost:4000/就能够看到我们配置的静态网页了

将本地Hexo部署到Github上

a.打开本地`_config.yml` 文件，在文件的末尾添加 
           # Deployment
           ## Docs: https://hexo.io/docs/deployment.html
           deploy:
             type: git
             repository: git@github.com:yinleiquan/yinleiquan.github.io.git
             branch: master
b.执行$ npm install hexo-deployer-git --save，安装git部署工具
c.执行$ hexo d，将本地文件刷新提交到Github上
到这一步，我们打开https://yinleiquan.github.io/网址就能看到我们部署上去的静态网页了

域名注册及解析

a.选用自己喜欢的服务商，推荐可以使用阿里云进行域名的注册和解析
b.回到本地的Hexo根目录下的 source文件夹中，新建一个文档文件，内容就是自己的域名，此处不需要加协议名，并且将这个文件更名为"CNAME"后缀也不要了。
c.对Github进行设置，选择自己项目的仓库，点击setting选项，进入到设置界面，在Custom domain出输入自己的域名，点击SAVE即可。
到这一步我们就已经可以访问http://yjlove.top/自己的域名博客了。
